# Claude Monitor 视频口播稿

> 预计时长：3-4 分钟
> 风格：技术分享，轻松自然，偏口语化

---

## 开场（0:00 - 0:20）

大家好，我是 XX。

你们平时用 Claude Code 写代码的时候，有没有这种感觉——开了好几个终端窗口，每个都在跑不同的任务，过了一会儿就分不清哪个在干活、哪个卡住了、哪个在等你输入。

特别是跑子代理的时候，你根本不知道它到底在干嘛，是还在思考，还是在改文件，还是已经结束了。

所以我就做了一个小工具，叫 **Claude Monitor**，一个 Claude Code 的实时监控面板。

---

## 演示总览（0:20 - 0:50）

先给大家看一眼效果。

（展示界面模拟/Hero 区域）

这就是面板的主界面。左边是进程列表，所有正在运行的 Claude Code 实例都会自动出现在这里。右边是选中进程的详细信息。

整个界面用的是复古像素风，有 CRT 扫描线特效，也能切换到浅色主题。

最重要的是——零配置，不需要改 Claude Code 的任何东西，直接打开就能用。

---

## 核心功能 1：实时状态（0:50 - 1:30）

（展示进程列表部分）

看左侧列表。每个进程的状态不是简单的「运行中」或「空闲」。

这里能看到三种具体状态：

- **EXECUTING**（绿色）—— 正在执行工具，比如在读写文件、运行命令
- **THINKING**（紫色，闪烁）—— 正在思考，还没开始动手
- **WAITING**（蓝色，闪烁）—— 等你输入，该你说话了

这对同时跑多个项目的同学特别有用。你不用挨个切到终端窗口去看，瞄一眼侧栏就知道哪个进程在忙、哪个在等你。

而且有新进程启动或者退出的时候，右上角会弹通知提醒你。

---

## 核心功能 2：Token 用量追踪（1:30 - 2:00）

（展示 Token 统计栏）

顶部这一行是全局的 Token 汇总。Input、Output、Cache 各用了多少，以及所有进程的总运行时长。

点进某个进程，下面还有这个会话自己的 Token 统计条，包括用了哪个模型。

如果你的 Claude Code 跑了一整天，想看看某个项目到底烧了多少 Token，这里一目了然。特别是 Cache 命中率很高的时候，能让你心里有个数。

---

## 核心功能 3：子代理监控（2:00 - 2:30）

（展示 Subagent 卡片）

这个是我自己最想要的功能。

Claude Code 用 Task 工具启动子代理的时候，之前你完全看不到子代理在干嘛。现在面板里会列出所有子代理：

- 每个 Agent 的 ID 和类型，比如 EXPLORE、PLAN
- 实时状态：RUNNING 还是 COMPLETED
- Token 消耗、消息数、执行时长、工具调用次数
- 如果还在跑，会显示它当前正在做什么操作

侧栏的进程条上也会有一个小标签，比如 `SA:1/4`，表示 4 个子代理里有 1 个还在跑。

子代理列表还可以折叠，不用的时候收起来就行。

---

## 核心功能 4：任务进度（2:30 - 3:00）

（展示进度条）

当 Claude Code 使用 TodoWrite 管理任务的时候，面板会自动生成一个进度条。

比如这里显示 3/6，50%，说明 6 个任务完成了 3 个。绿色是已完成的，黄色闪烁的是正在进行中的。

下面还有未完成任务的具体列表，每个任务的名称都展示出来了。

有了这个，长时间运行的任务你就不用一直盯着了，看一眼进度条就知道大概完成了多少。

---

## 核心功能 5：日志和工具箱（3:00 - 3:20）

（简要展示）

日志区可以看到最近的对话记录，按角色分色标记。工具调用可以展开看详情，Edit 操作还能看到 old/new 的 diff 对比。

顶部的 TOOLKIT 按钮点开，能看到当前会话可用的所有 Skills 和 MCP 工具。

---

## 怎么用（3:20 - 3:40）

使用方式非常简单。

确保有 Python 3.10 以上和 Node.js 18 以上。

Windows 双击 `start.bat`，Linux 运行 `./start.sh`。或者直接 `python run.py`，它会自动装依赖、构建前端、启动服务，然后打开浏览器。

全程不需要任何配置。
